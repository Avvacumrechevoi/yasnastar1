# Архитектура расширения интерфейса Ясны

## Общая идея

После утверждения разводящего главного экрана интерфейс должен развиваться не в сторону перегруженного дашборда, а в сторону **цепочки понятных продуктовых экранов**. Главный экран остаётся точкой входа, а дальше пользователь уходит либо в **рабочую карту «Звезда Ясны»**, либо в **каталог всех Ясн**, где можно выбрать нужную модель по типу.

## Новая навигационная схема

| Маршрут | Роль | Что пользователь получает |
|---|---|---|
| `/` | Разводящий лендинг | Центральный оффер, вход в метод, переходы к инструментам и новым разделам |
| `/star` | Экран «Звезда Ясны» | Интерактивное ядро переговорной аналитики со всеми механиками |
| `/yasnas` | Каталог всех Ясн | Полный перечень Ясн с фильтрацией по типу и быстрым обзором полочек |

## Страница «Звезда Ясны»

Экран строится как **аналитический навигационный стол** из трёх зон. Слева располагается контекст кейса, по центру — интерактивная звезда, справа — инспектор выбранной механики. Внизу идёт лента этапов Ясны, чтобы связать карту с методикой без тяжёлой энциклопедии.

| Зона | Содержимое | Поведение |
|---|---|---|
| Левая колонка | Название кейса, цель встречи, сигналы, цитаты, статус переговоров | Статична, но обновляет акценты в зависимости от выбранной механики |
| Центральная сцена | Ядро Ясны, 8 механик по кругу, линии связей, переключатели слоёв | Hover даёт краткий preview, click делает механику активной |
| Правая колонка | Смысл механики, что она отслеживает, артефакты, следующий шаг | Меняется по активному узлу |
| Нижняя лента | Этапы: архив, отражение, связи, выбор, мост, радуга, результат | Подсвечивает, на каких этапах сильнее работает выбранная механика |

В звезду входят восемь рабочих механик: **Архив кейса, Поток диалога, Карта участников, Точки рычага, История уступок, Оценка риска, Путь соглашения, Горизонт результата**. Дополнительно на первом макете должны быть мягкие переключатели слоёв: **Сигналы, Влияние, Уступки, Риски**.

## Страница всех Ясн

Каталог должен восприниматься не как таблица, а как **живая библиотека моделей Ясны**. Сверху располагается краткий оффер и панель фильтрации. Ниже — адаптивная сетка карточек. На карточке пользователь видит название Ясны, тип, краткое объяснение и несколько характерных полочек как preview-семантику.

### Предлагаемая модель фильтрации

| Тип | Какие Ясны входят |
|---|---|
| Пространственные | Ясна Дома, Ясна Спальни, Ясна Кухни, Ясна Двора, Ясна Колокольни, Ясна Театра |
| Природные | Треугольники Элементов, Ясна Цветов, Ясна Суток, Ясна Круговорота Воды, Ясна Дерева, Ясна Мирового Яйца, Ясна Мирового Яйца 2 |
| Телесные | Ясна головы, Ясна Тела, Ясна внут.органов |
| Живые и бытовые | Ясна Двора (Животных), Ясна Печи, Ясна Удочки |
| Системные и социальные | Ясна Заода (+Предприятия), Знаки З., Ясна месяцев |

Фильтрация должна работать без перезагрузки страницы. При выборе типа сетка карточек сразу перестраивается, а активный фильтр остаётся визуально выделенным. Хорошо также добавить счётчик найденных Ясн и кнопку сброса фильтра.

## Связь между экранами

Главный экран должен получить два явных входа в новые разделы: один в хедере и один в секции инструментов/методов. Страница «Звезда» и каталог Ясн должны использовать **один и тот же визуальный язык**: тёмный фон, чистая типографика, зелёный акцент, умеренная анимация, лёгкие свечения и стеклянные панели. При этом экран звезды должен быть более сценическим и центральным, а каталог — более структурным и библиотечным.

## Решение по сценариям пользователя

Пользователь может входить в продукт двумя путями. Первый путь — через понимание метода: с лендинга он идёт в «Звезду Ясны» и видит, как устроена переговорная механика. Второй путь — через изучение инструментария: он открывает каталог всех Ясн, фильтрует нужный тип и выбирает модель для дальнейшего освоения.

Обе страницы должны поддерживать быстрый возврат на главную, чтобы разводящий экран не терял своей роли навигационного центра.
