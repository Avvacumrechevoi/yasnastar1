# Визуальная проверка центральной сцены «Звезда»

## Что видно сейчас

| Наблюдение | Вывод для правки |
| --- | --- |
| Центральная звезда расположена слишком низко в сцене, над ней остаётся много пустого пространства | Поднять геометрию и увеличить полезный масштаб SVG, чтобы композиция была собрана по центру |
| Линии и слои активных механик выглядят слишком плотными и частично перекрывают читаемость внешних подписей | Нужно пересобрать приоритет слоёв и снизить шум вторичных механик |
| Внешние подписи полочек местами визуально оторваны от круга | Нужна более строгая привязка labelPoint и единая логика смещений |
| Центральная подпись Ясны должна остаться компактной и не перекрывать активную точку | Не возвращать крупную карточку в центр, оставить только лаконичный центр |
| Нижняя служебная строка «6 наверху · 0 внизу» уже не нужна | Не выводить её повторно |
| При активной механике фокус должен читаться так: выбранная точка — главный акцент, активная механика — второй слой, остальные механики — тихий контекст | В коде стоит усилить визуальную иерархию состояний |

## Следующая цель

Пересобрать геометрию центральной сцены так, чтобы звезда была крупнее, выше и чище, а сочетание состояний «выбранная полочка / активная механика / прочие механики / все механики» не конфликтовало визуально.

## Дополнительная проверка после правок

| Наблюдение | Что ещё нужно поправить |
| --- | --- |
| Звезда стала чище: вторичные механики меньше спорят с активным треугольником | Логику ослабления фоновых слоёв можно оставить |
| Подписи полочек стали компактнее, центральная карточка не вернулась | Это соответствует требованиям и не требует отката |
| Внутри центральной сцены остаётся неидеальный вертикальный баланс: на одном экране звезда визуально сидит ниже, на другом — видно, что у композиции остаётся слишком много пустоты снизу | Нужно не просто сдвигать SVG, а точнее поджать область просмотра, чтобы вся геометрия помещалась плотнее и крупнее |
| Полная звезда всё ещё читается как слишком растянутая по высоте сцены относительно окна браузера | Нужна более жёсткая настройка области просмотра SVG, а не только translate |

## Техническая находка по реальной причине перекоса

| Факт | Значение |
| --- | --- |
| Реальный размер SVG в браузере | 840 × 1067 px |
| Ожидаемая форма | Квадратная сцена 1:1 |
| Следствие | SVG растягивается по высоте и искажает восприятие: звезда уходит вниз/вверх в зависимости от положения окна |
| Корневая причина | Класс SVG одновременно тянет ширину и высоту контейнера (`h-full w-full`), из-за чего поле перестаёт быть квадратным |
| Нужная правка | Зафиксировать квадратную область сцены (`aspect-square`, `h-auto`) и масштабировать звезду по ширине, а не по высоте |

## Проверка после фикса квадратного SVG

| Наблюдение | Вывод |
| --- | --- |
| Поле сцены больше не растягивается по высоте | Корневая техническая ошибка устранена |
| В первом экране звезда видна слишком низко и слишком мелко внутри центральной панели | Теперь нужно поднять и масштабировать сам квадратный SVG внутри сцены |
| Пустое пространство в верхней части центральной панели остаётся чрезмерным | Нужен управляемый отрицательный сдвиг и более крупный `max-width` для квадратного поля |

## Проверка сочетаний состояний точки и механики

| Сценарий | Наблюдение | Вывод |
| --- | --- | --- |
| Выбор полочки `6 · Перенос` при активной механике `Огонь ХА` | Активная логика остаётся у механики, но текущая точка переключается на `6 · Перенос`; по смыслу это состояние «точка вне активного слоя» | Комбинация «выбранная точка вне ведущей механики» поддерживается и не ломает сцену |
| Переключение активной механики на `Воздух ЦИ` для той же полочки `6` | Активный слой меняется на `Воздух ЦИ`, счётчик выбранных механик увеличивается до `2 из 11`, а выбранная точка остаётся прежней | Комбинация «выбранная точка входит в активный слой» работает корректно и сохраняет приоритет выбранной точки |
| Общий режим `11/11` при выбранной полочке `6 · Перенос` и активной механике `Воздух ЦИ` | Переключатель показывает `Снять все механики 11/11`; мультивыбор включён, но активная механика остаётся `Воздух ЦИ`, а выбранная полочка не сбрасывается | Комбинация «все механики + активная механика + выбранная точка» ведёт себя устойчиво: обзор расширяется, но точка не теряет приоритет |
| Финальная проверка первого экрана | Звезда видна целиком без перекрывающей центральной карточки; боковые панели стали компактнее, а в центре читаются только короткие маркеры состояния (`Атмосфера`, `6 · Перенос`, `Воздух ЦИ`) | Композиция стала чище: фокус смещён на саму звезду, служебные блоки больше не перетягивают внимание |
