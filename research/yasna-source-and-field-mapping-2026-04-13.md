# Источник данных и матрица полей для интерфейса Ясны

## Происхождение данных

Текущий прикладной набор данных для базовых Ясн берётся не из разрозненных исходников напрямую во время рендера, а из уже подготовленного файла `research/yasna_dataset.json`. Этот JSON является нормализованным слоем поверх извлечённых материалов из папки `research/upload_lessons/`.

Индекс `research/upload_lessons/index.md` подтверждает происхождение содержимого: в нём собраны расшифровки и конспективные извлечения по исходным урокам (`.docx` и связанным с ними видео/аудио-материалам). На видимом фрагменте индекса явно присутствуют уроки по **Ясне суток** и **Ясне домашних животных**, а сама структура проекта и `yasna_dataset.json` показывают, что нормализация затем была продолжена для пяти базовых Ясн.

Практически это означает следующее: для серверной модели источником runtime-данных нужно считать `yasna_dataset.json`, а `upload_lessons/index.md` и связанные извлечённые тексты — источником происхождения и верификации.

## Базовые записи, уже присутствующие в `yasna_dataset.json`

| ID | Заголовок | Кол-во уроков | Основной предметный фокус |
| --- | --- | ---: | --- |
| `суток` | Ясна суток | 1+ | Световой цикл суток, горизонт, видимая и скрытая часть пути Солнца |
| `домашних-животных` | Ясна домашних животных | 5 | Хозяйственный двор, постройки, животные и бытовая организация пространства |
| `круговорота-воды` | Круговорот воды | 5 | Природный цикл воды, поверхностный и подземный контуры, противоположные точки |
| `облачности` | Облачность | 1 | Цикл сгущения и рассеивания облачности, просветы и ясность |
| `цветов-радуги` | Цвета радуги | 1 | Двенадцатичастная палитра, стихии, противоположности и переходные оттенки |

## Какие поля датасета существуют у каждой записи

Текущий тип записи в `server/yasna/catalog.ts` совпадает с реальной структурой базового датасета:

| Поле | Назначение | Уже используется в серверном каталоге | Уже доходит до UI |
| --- | --- | --- | --- |
| `id` | Стабильный идентификатор вкладки/режима | Да | Да |
| `family` | Семейство Ясны | Да | Частично |
| `title` | Короткий заголовок | Да | Да |
| `lesson_count` | Счётчик уроков | Да | Частично |
| `summary` | Краткое описание Ясны | Да | Да |
| `mechanics` | Список механик/осей/контрастов | Да | Да |
| `points` | 12-позиционная семантика полочек | Да | Да |
| `lessons` | Нормализованные уроки с метаданными | Да | Частично |
| `notes` | Концентрированные интерфейсные/методические заметки | Да | Частично |

## Структура `lessons`

Каждый урок внутри записи несёт уже достаточно богатое содержание для server-driven интерфейса:

| Поле урока | Что содержит | Потенциал для интерфейса | Текущий статус использования |
| --- | --- | --- | --- |
| `file` | Имя нормализованного файла-источника | Трассировка происхождения и ссылки на источник | Пока почти не показывается |
| `lesson` | Название урока | Навигация по урокам / заголовки карточек | Используется частично |
| `topics` | Сжатое содержание урока | Карточки «о чём урок» / инспектор / правые панели | Используется ограниченно |
| `point_assignments` | Явное сопоставление полочек и смыслов | Overlay-инспектор, подсказки по точкам, быстрые карточки чтения | Уже начал использоваться |
| `mechanics_mentions` | Какие механики упоминаются в уроке | Подсветка связей между точками и механиками | Используется частично |
| `interface_notes` | Подсказки, как это показывать в UI | Готовый источник для проектных карточек и объяснений | Уже начал использоваться |

## Матрица сопоставления вкладок с содержимым

| Вкладка / Ясна | Что уже хорошо заполнено в датасете | Что уже используется в текущем интерфейсе | Что пока недоиспользуется |
| --- | --- | --- | --- |
| **Суток** | Цикл света/тьмы, горизонт, переломные точки, временные этапы | Базовые точки, summary, механики | Уроковые выдержки, явные `point_assignments`, интерфейсные заметки |
| **Домашних животных** | Пространство двора, дом, сад, околица, хозяйственные зоны | Базовая вкладка и 12-точечный контур | Богатые уроковые объяснения и привязка объектов двора к точкам |
| **Круговорот воды** | Самая насыщенная запись: 5 уроков, двойной цикл, колодец, животные, противоположности | Точки, summary, часть notes | Многоуровневые lesson-карточки, отдельные сценарии по противоположностям и колодцу |
| **Облачность** | Чистая и очень пригодная для атмосферы схема: покров, просветы, ясность, обратное сгущение | Используется как база для переговорной Ясны атмосферы | Базовую природную версию можно раскрыть сильнее через notes и lesson-card |
| **Цвета радуги** | Полная 12-позиционная палитра, стихии и предельные состояния | Базовая вкладка существует | Цветовые и сравнительные режимы почти не раскрыты в UI |

## Связь базового датасета и переговорного overlay

Сейчас серверная модель устроена в два слоя:

| Слой | Где хранится | Что содержит |
| --- | --- | --- |
| Базовый слой | `research/yasna_dataset.json` | 5 исходных Ясн, извлечённых из уроков |
| Переговорный overlay | `server/yasna/catalog.ts` | 4 прикладные переговорные Ясны: атмосфера встречи, облака проблем, образ проблемы, решения |

Это хорошая архитектура: базовые Ясны остаются привязанными к учебному материалу, а переговорные режимы живут как отдельный server-driven прикладной слой поверх датасета.

## Что уже доведено до пользователя

На текущем этапе в интерфейс уже доходят:

| Канал | Что реально показывается |
| --- | --- |
| Основная сцена `/star` | Заголовки, summary, точки полочек, механики |
| Overlay-инспектор | Точка, механика, а также дополнительные real-data карточки из уроков |
| Переговорный режим | 4 server-driven переговорные Ясны и слой атмосферных примет/предсказаний |

## Что пока остаётся недоиспользованным

| Нераскрытый ресурс | Почему это важно |
| --- | --- |
| Полные `topics` из уроков | Это готовые микро-конспекты для правой панели или вкладки «Уроки» |
| Все `mechanics_mentions` по урокам | Можно лучше связывать выбор полочки с активными механиками и дугами |
| `file` как provenance | Пользователю и исследователю полезно видеть, откуда пришла конкретная формулировка |
| Сводные `notes` по каждой Ясне | Это почти готовый слой «как читать эту Ясну в интерфейсе» |
| Богатые многoурочные записи (`круговорота-воды`, `домашних-животных`) | Пока UI забирает только небольшую часть их глубины |

## Рекомендуемый следующий практический шаг

Следующий осмысленный шаг — **не добавлять новые пустые Ясны**, а усилить уже существующие server-driven экраны за счёт реального lesson-контента из базового датасета.

Приоритет логично держать таким:

| Приоритет | Шаг |
| --- | --- |
| 1 | Добавить в правую колонку `/star` компактный блок «Уроки по этой Ясне» на основе `lessons[].topics` |
| 2 | Показывать происхождение карточки или формулировки через `lesson` и `file` |
| 3 | Связать `mechanics_mentions` урока с подсветкой механик и объясняющих карточек |
| 4 | Для насыщенных Ясн (`круговорота-воды`, `домашних-животных`) сделать разворачиваемые lesson-notes вместо одного краткого summary |

## Вывод

Аналитический пробел закрывается так: данные уже **извлечены и нормализованы**, а задача теперь заключается не в повторном извлечении, а в **дальнейшем поэтапном доведении богатых полей `yasna_dataset.json` до интерфейса**. Текущий runtime-источник — `yasna_dataset.json`, а документальная трассировка происхождения обеспечивается через `research/upload_lessons/index.md` и имена файлов в `lessons[].file`.
