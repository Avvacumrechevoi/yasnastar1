# Архитектура нового главного экрана Ясны

## Роль экрана

Главный экран больше не играет роль рабочего дашборда. Он становится разводящим входом в метод Ясны: сначала объясняет ценность, затем предлагает выбрать инструмент, а уже потом ведёт в более глубокие разделы.

## Композиция страницы

| Зона | Содержание | Поведение |
|---|---|---|
| Хедер | Логотип Ясны слева, короткая навигация справа, спокойная кнопка входа в сценарий | При клике пункты ведут к якорным секциям страницы |
| Hero | Крупный оффер по центру, короткое пояснение, основной и вторичный CTA | Фокус на одном главном обещании: освоить Ясну и увереннее вести переговоры |
| Визуальные акценты | 2–3 плавающих превью-карточки вокруг оффера | Лёгкая анимация, не мешающая чтению текста |
| Пояс преимуществ | Три коротких тезиса под hero | Быстро объясняют, зачем нужен метод |
| Секция инструментов | Сетка из кликабельных и анимированных карточек | Карточки ведут к соответствующим секциям ниже страницы |
| Детальные секции | Для каждого инструмента отдельный блок с описанием, вопросом и действием | Это и есть целевые разделы, куда попадает пользователь по клику |
| Нижний CTA | Финальный вход в сценарий изучения метода | Закрывает страницу простым действием |

## Состав инструментов на первом экране

| Инструмент | Роль в структуре |
|---|---|
| Атмосфера | Помогает увидеть общее поле разговора |
| Облака | Собирает напряжения и узлы переговоров |
| Образы | Помогает различать характер проблемы |
| Решения | Переводит анализ в следующий шаг |
| Синтез | Сопоставляет две Ясны и собирает общую картину |
| Кейсы | Показывает, как метод применяется в реальных переговорах |

## Правила визуального языка

Экран должен быть тёмным, графитовым и спокойным. Акцент — насыщенный зелёный, как маркер действия и движения. Заголовок крупный, но без декоративной сложности. Карточки инструментов — компактные, с ровной геометрией, мягким свечением и умеренным hover-движением. Вся страница должна выглядеть как уверенный продуктовый вход, а не как арт-эксперимент.

## Поведение анимации

Hero-превью плавают медленно и асинхронно. Карточки инструментов слегка приподнимаются при наведении, подсвечивают контур и усиливают иконку. Переходы по якорям плавные. Анимация должна создавать ощущение живого продукта, но не мешать чтению и выбору.
